# Atlas Quiz

An interactive geography quiz built on real country shapes. Pick the
world map or one continent, guess the highlighted country by name,
zoom/pan to explore, and a "neighborhood" inset kicks in automatically
for countries too small to make out at continent scale.

## Project structure

```
atlas-quiz/
  src/
    AtlasQuiz.jsx     — the entire app (single component, ~190KB with
                         the embedded country-shape data)
    entry.jsx         — mounts <AtlasQuiz /> into #root
  public/              — static site root; this is what you deploy
    index.html
    manifest.json      — PWA install metadata
    sw.js              — offline service worker
    icons/
    app.bundle.js       — generated by `npm run build`, not committed
  data/
    build-data.js       — regenerates data/countries.json from Natural
                           Earth data (see comment at top of the file
                           for the exact pipeline)
    countries.json       — standalone copy of the country dataset
                           (the same data is also embedded directly in
                           src/AtlasQuiz.jsx as the COUNTRIES constant)
  scripts/
    build.js            — production bundle (esbuild, minified)
    dev-server.js       — local dev server with live rebuild
```

## Setup

```
npm install
```

## Local development

```
npm run dev
```

Opens a dev server (default http://localhost:8080) that rebuilds
`src/entry.jsx` on every request. Edit `src/AtlasQuiz.jsx` and refresh.

## Production build

```
npm run build
```

Bundles everything (React, ReactDOM, D3, the app, and the country
data) into `public/app.bundle.js`. Deploy the `public/` folder as a
static site — every path inside it is relative, so it works from any
subdirectory or static host (Netlify, Vercel, GitHub Pages, S3, nginx,
whatever).

The `public/` folder is also a valid installable PWA as-is: open it in
Safari on iPhone and use Share → Add to Home Screen.

## Regenerating the country data

```
npm run build-data
```

Rebuilds `data/countries.json` from `world-atlas` + `world-countries`
(both installed as devDependencies). Read the comment at the top of
`data/build-data.js` for what the pipeline actually does — it's not
just "download a geojson file," there's real logic in there for
merging split territories, dropping colonial exclaves, and keeping
tiny nations from collapsing to zero-area shapes during simplification
(these were all real bugs at some point, hence the guardrails).

If you regenerate it, copy the new `data/countries.json` contents into
the `COUNTRIES` constant near the top of `src/AtlasQuiz.jsx` — the app
reads from that inline constant, not from the JSON file, so it stays a
single self-contained bundle with no runtime fetch.

## Stack

- React 19 (function components + hooks, no other framework)
- D3 (`d3-geo` for projections/paths, `d3-zoom` for the pan/zoom map)
- esbuild for bundling — no webpack/vite config to fight with
- No CSS framework; all styles are inline in `AtlasQuiz.jsx` via a
  `<style>` tag, using CSS variables for the color palette

## Notes on the map implementation

A few things worth knowing if you're modifying `AtlasQuiz.jsx`:

- **Projection per scope**: each continent gets its own aspect ratio
  and a projection rotation centered on that region's mean longitude,
  so continents that straddle the antimeridian (Oceania, or Europe
  once Russia's Far East is included) don't blow out the bounding box
  and zoom everything out to nothing.
- **Small-country handling**: if the highlighted country is too small
  to read at the current zoom (`isSmallBase`/`isSmallLive` in
  `CountryMap`), a locator dot + dashed "you'll see this area" box
  appear on the main map, and a separate zoomed inset renders the
  country properly with its real neighbors. The main map is also
  itself zoomable (`d3-zoom`), and if you zoom in far enough manually,
  the dot disappears and the real shape highlights normally.
- **Answer checking**: `checkAnswer()` normalizes input (strips
  diacritics/articles), checks an alias table for common alternate
  names (USA, UK, DR Congo, ...), and falls back to a small Levenshtein
  distance tolerance for typos.
